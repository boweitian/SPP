# SPP — Code for the Main-Paper Figures

This repository contains the cleaned code used to generate the two main-paper
figures on Spectral Principal Path (SPP):

- **Figure 1 — Spectral Dominance** (`spp_results/multi_model/fig1_spectral_dominance.pdf/png`)
- **Figure 2 — Context Incoherence** (`spp_results/multi_model/fig2_context_incoherence.pdf/png`)

The package has been trimmed to the components that actually produce these two
figures. Exploratory experiments that did not appear in the paper have been
removed so the pipeline runs end-to-end without dead dependencies.

## Pipeline overview

The scripts are prefixed `A_`, `B_`, `C_`, `D_` and run in that order:

| Step | Script | Input | Output |
|------|--------|-------|--------|
| A | `A_train_get_captions.py` | COCO val2017 images | `file/coco.json` |
| B | `B_process_initial_dataset.py` | `file/coco.json` | `file/coco_honesty_dataset.json` |
| C | `C_spp_experiments.py` | `file/coco_honesty_dataset.json` | per-model JSON in `spp_results/<model>/` |
| D | `D_multi_model_plots.py` | per-model JSON | `spp_results/multi_model/fig1*, fig2*` |

`C` is run once per model; `D` then aggregates across all models into the two
combined figures.

## How COCO is used

The dataset is **COCO val2017**, and we use it only as a source of **images**,
not its caption annotations. The text analysed by the method is generated by the
model, not taken from COCO:

1. **A** loads COCO val2017 images and, for each image, prompts a
   vision-language model (`HuggingFaceM4/idefics2-8b`) with the instruction
   *"Pretend you're an honest person making statements about the world."* The
   model's free generation becomes the "statement" for that image. Statements
   are saved with their COCO `image_id` to `file/coco.json`.
2. **B** tokenizes each statement and expands it into many **token-prefix
   truncations** (one sample per prefix). Each truncation is paired into a
   contrastive *honest* / *untruthful* prompt that shares the same image and
   prefix, differing only in the persona word. This is why a small set of
   statements becomes a much larger set of contrastive pairs in
   `file/coco_honesty_dataset.json`.
3. **C** runs each prompt pair through the analysed LLM, collects per-layer
   hidden states, and forms the per-sample concept-difference matrix
   `D^(l) = h_honest^(l) − h_untruthful^(l)` used by the SPP metrics.

In short, COCO provides a diverse, fixed set of visual contexts that elicit
varied model statements; the honest/untruthful contrast is imposed by the prompt
template, not by COCO labels.

## Output JSON produced by C

`C_spp_experiments.py` writes two files per model into
`spp_results/<model_slug>/`:

- `expA_enhanced_spectral_gap.json` — spectral gap rho = sigma_1/sigma_2 of the
  difference matrix, a shuffled-label baseline, cumulative amplification, and
  concept alignment. Consumed by **Figure 1**.
- `expB_context_incoherence.json` — per-layer individual vs. collective
  perturbation magnitudes and the diffuseness ratio. Consumed by **Figure 2**.

`D_multi_model_plots.py` reads exactly these two files for every model in its
`MODELS` dictionary and produces the combined `fig1`/`fig2`.

## Reproducing the figures

### Requirements

```
python >= 3.9
torch, transformers, numpy, scipy, matplotlib, seaborn, tqdm
```

A CUDA GPU is recommended for steps A and C (model forward passes). They run on
CPU but slowly.

### Run order

```bash
# Step A: generate statements from COCO val2017 images (requires the COCO images
#         and the dataset loader; see "External inputs" below). Produces file/coco.json.
python A_train_get_captions.py

# Step B: build the honest/untruthful contrastive dataset.
python B_process_initial_dataset.py

# Step C: run the SPP experiments. Run ONCE PER MODEL.
python C_spp_experiments.py --model Qwen/Qwen2.5-7B
python C_spp_experiments.py --model meta-llama/Llama-3.1-8B-Instruct
python C_spp_experiments.py --model mistralai/Mistral-7B-Instruct-v0.3
python C_spp_experiments.py --model cognitivecomputations/Wizard-Vicuna-30B-Uncensored
# Each call writes to spp_results/<model_slug>/.
# The slug (model name after the last "/", lowercased) must match the keys in
# D_multi_model_plots.py's MODELS dict:
#   qwen2.5-7b, llama-3.1-8b-instruct, mistral-7b-instruct-v0.3,
#   wizard-vicuna-30b-uncensored

# Step D: aggregate across models into the two main-paper figures.
python D_multi_model_plots.py
# Writes spp_results/multi_model/fig1_spectral_dominance.{pdf,png}
#    and spp_results/multi_model/fig2_context_incoherence.{pdf,png}
```

### Notes / caveats

- **Pre-generated intermediates are included.** `file/coco.json` and
  `file/coco_honesty_dataset.json` ship with the package, so steps **C** and
  **D** can be run without re-running A and B. The final figures in
  `spp_results/multi_model/` are also included.
- **Per-model result directories are not shipped.** The
  `spp_results/<model_slug>/` directories that D reads from are produced by
  running C on each model; they are not bundled. Run C on the four models above
  before running D, or edit D's `MODELS` dict to match the models you have run.
- **External inputs for step A.** `A_train_get_captions.py` expects COCO val2017
  images under `data/COCO/val2017` and imports a dataset loader
  (`from dataset import COCOVal2017Dataset, transform`). The COCO images and that
  loader are standard external resources and are not redistributed here. To
  reproduce from scratch, supply them; otherwise skip A/B and use the provided
  `file/*.json`. Step A uses `temperature=2.0` during generation to encourage
  diverse statements (intentional).
- **`file/processed_coco_honesty_dataset.json`** is a pre-tokenized variant kept
  for reference; it is not read by C or D and is not required for the figures.
- **`--only-plots` flag in C is not part of this pipeline.** It refers to an
  earlier standalone plot that depended on a file not included here; use the
  default invocation (no flag) as shown above.
